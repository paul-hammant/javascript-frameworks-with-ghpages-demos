content" id="content">\n     <a href="index.html">\n      <img id="logo" src="img/logo.png"/>\n     </a>\n     <h1 id="h-backbonegiraffe">\n      Backbone.Giraffe\n     </h1>\n     <h2 id="h-introduction">\n      Introduction\n     </h2>\n     <p>\n      <a href="http://barc.github.io/backbone.giraffe">\n       <strong>\n        Backbone.Giraffe\n       </strong>\n      </a>\n      is a light and flexible library that extends\n      <a href="http://documentcloud.github.com/backbone/">\n       <strong>\n        Backbone.js\n       </strong>\n      </a>\n      to new heights.\n      <strong>\n       Giraffe\n      </strong>\n      \'s goal is to follow the\n      <strong>\n       Backbone\n      </strong>\n      philosophy of unopinionated\nsimplicity to provide commonly needed features with few assumptions. It differs\nfrom other\n      <strong>\n       Backbone\n      </strong>\n      libraries like\n      <strong>\n       Marionette\n      </strong>\n      and\n      <strong>\n       Chaplin\n      </strong>\n      in its\nreduced scope and size, and it takes a different approach to the problems of\nroute handling, object lifecycles, event aggregation, and view management.\n     </p>\n     <h2 id="h-overview">\n      Overview\n     </h2>\n     <ul>\n      <li>\n       <p>\n        <strong>\n         Giraffe.View\n        </strong>\n        is a nestable and flexible class that provides lifecycle\nmanagement and many other useful features. It defaults to\n        <strong>\n         Underscore\n        </strong>\n        templates and easily supports any form of templating.\n       </p>\n      </li>\n      <li>\n       <p>\n        <strong>\n         Giraffe.App\n        </strong>\n        is a special view that helps your views, models, collections,\nand routers communicate by acting as an event hub. Multiple apps can coexist and\nteardown is as simple as it gets.\n       </p>\n      </li>\n      <li>\n       <p>\n        <strong>\n         Giraffe.Router\n        </strong>\n        leverages an app\'s events to trigger routing events that\nany object can listen for. It also has reverse routes to allow the construction\nof URLs using app events and arguments.\n       </p>\n      </li>\n      <li>\n       <p>\n        <strong>\n         Giraffe.Model\n        </strong>\n        and\n        <strong>\n         Giraffe.Collection\n        </strong>\n        are thin wrappers that add\n        <strong>\n         Giraffe\n        </strong>\n        \'s lifecycle management and app events. Any object can mix\nin this functionality via\n        <code>\n         Giraffe.configure\n        </code>\n        .\n       </p>\n      </li>\n     </ul>\n     <h2 id="h-documentation">\n      Documentation\n     </h2>\n     <p>\n      Read the\n      <a href="http://barc.github.io/backbone.giraffe/backbone.giraffe.html">\n       <strong>\n        API docs\n       </strong>\n      </a>\n      and\ncheck out our\n      <a href="http://barc.github.io/backbone.giraffe/viewBasics.html">\n       <strong>\n        live examples\n       </strong>\n      </a>\n      .\n     </p>\n     <h2 id="h-how-giraffe-is-different">\n      How Giraffe is Different\n     </h2>\n     <p>\n      <strong>\n       Giraffe\n      </strong>\n      was created by the needs of our team as we built\n      <a href="http://barc.com">\n       <strong>\n        Barc\n       </strong>\n      </a>\n      . We tried many existing libraries but some did way too\nmuch, others added too many layers, and others performed poorly.\n     </p>\n     <p>\n      <strong>\n       Giraffe\n      </strong>\n      does not have all the bells and whistles of the larger frameworks.\nWe found the effort to customize them for our needs was more effort than simply\nbuilding upon\n      <strong>\n       Backbone\n      </strong>\n      with a minimalist approach. For example, there is no\nconcept of specialized containers like regions or layouts, as any view in\n      <strong>\n       Giraffe\n      </strong>\n      can act as a parent of one or more child views.\n      <strong>\n       Giraffe\n      </strong>\n      also\nhas no CollectionView or ItemView\n(see\n      <a href="http://barc.github.io/backbone.giraffe/collectionView.html">\n       <strong>\n        Giraffe.Contrib.CollectionView\n       </strong>\n      </a>\n      and\n      <a href="http://barc.github.io/backbone.giraffe/fastCollectionView.html">\n       <strong>\n        Giraffe.Contrib.FastCollectionView\n       </strong>\n      </a>\n      ),\nbut we are open to suggestions to make\n      <strong>\n       Giraffe\n      </strong>\n      as useful as possible to\n      <strong>\n       Backbone\n      </strong>\n      developers who want an extension library with few opinions.\n     </p>\n     <p>\n      Is this framework for you? It depends. We feel\n      <strong>\n       Giraffe\n      </strong>\n      adds essential\nfeatures to make you more productive with\n      <strong>\n       Backbone\n      </strong>\n      .\n     </p>\n     <h3 id="h-highlights">\n      Highlights\n     </h3>\n     <ul>\n      <li>\n       <p>\n        <a href="http://barc.github.io/backbone.giraffe/routersAndAppEvents.html">\n         <strong>\n          Routes emit events\n         </strong>\n        </a>\n        instead of being tied to functions. This makes it extremely simple for a deeply\nnested view to act on a route.\n       </p>\n      </li>\n      <li>\n       <p>\n        <a href="http://barc.github.io/backbone.giraffe/backbone.giraffe.html#Router">\n         <strong>\n          Reverse routes with arguments\n         </strong>\n        </a>\n        provide a way to trigger routes in the application using app events without\nhaving to know a URL path.\n       </p>\n      </li>\n      <li>\n       <p>\n        <a href="http://barc.github.io/backbone.giraffe/appEvents.html">\n         <strong>\n          Giraffe.App\n         </strong>\n        </a>\n        is a\nspecial view that acts as an event hub to help your app communicate and respond\nto routes, and all\n        <strong>\n         Giraffe\n        </strong>\n        objects have convenient\n        <code>\n         appEvents\n        </code>\n        bindings\ninspired by\n        <code>\n         Backbone.View#events\n        </code>\n        .\n       </p>\n      </li>\n      <li>\n       <p>\n        <a href="http://barc.github.io/backbone.giraffe/backbone.giraffe.html#View-attachTo">\n         <strong>\n          <code>\n           Giraffe.View#attachTo(someElement)\n          </code>\n         </strong>\n        </a>\n        allows views to move anywhere on the DOM without clobbering each other\'s events,\nand it automatically sets up parent-child relationships for memory management.\n       </p>\n      </li>\n      <li>\n       <p>\n        <a href="http://barc.github.io/backbone.giraffe/lifecycleManagement.html">\n         <strong>\n          Lifecycle management\n         </strong>\n        </a>\n        mitigates memory leaks. It\'s automatic for nested views and can be used for any\nobject with a\n        <code>\n         dispose\n        </code>\n        method via\n        <code>\n         Giraffe.View#addChild\n        </code>\n        .\n       </p>\n      </li>\n      <li>\n       <p>\n        <a href="http://barc.github.io/backbone.giraffe/appInitialization.html">\n         <strong>\n          (A)sync app initialization\n         </strong>\n        </a>\n        helps an app reach its ready state. For example, an app may need to wait for\nasynchronous bootstrap data or a websocket connection before starting.\n       </p>\n      </li>\n      <li>\n       <p>\n        <a href="http://barc.github.io/backbone.giraffe/documentEvents.html">\n         <strong>\n          Declarative event handling\n         </strong>\n        </a>\n        in markup provides simple one-way binding. (does not try to be\n        <strong>\n         Knockout\n        </strong>\n        or\n        <strong>\n         AngularJS\n        </strong>\n        )\n       </p>\n      </li>\n      <li>\n       <p>\n        <a href="https://github.com/barc/backbone.giraffe/blob/master/src/backbone.giraffe.contrib.coffee">\n         <strong>\n          Optional Giraffe.Contrib\n         </strong>\n        </a>\n        extensions include a simple Controller class with\n        <strong>\n         Giraffe\n        </strong>\n        \'s lifecycle and\n        <code>\n         appEvents\n        </code>\n        features,\na\n        <a href="http://barc.github.io/backbone.giraffe/collectionView.html">\n         CollectionView\n        </a>\n        that renders a collection with a view per model,\nand a\n        <a href="http://barc.github.io/backbone.giraffe/fastCollectionView.html">\n         FastCollectionView\n        </a>\n        that renders a collection with a single view\n        <a href="http://jsperf.com/collection-views-in-giraffe-and-marionette/5">\n         more efficiently than Giraffe.Contrib.CollectionView and Marionette.CollectionView\n        </a>\n        .\n content" id="content">\n     <a href="index.html">\n      <img id="logo" src="img/logo.png"/>\n     </a>\n     <h1 id="h-backbonegiraffe">\n      Backbone.Giraffe\n     </h1>\n     <h2 id="h-introduction">\n      Introduction\n     </h2>\n     <p>\n      <a href="http://barc.github.io/backbone.giraffe">\n       <strong>\n        Backbone.Giraffe\n       </strong>\n      </a>\n      is a light and flexible library that extends\n      <a href="http://documentcloud.github.com/backbone/">\n       <strong>\n        Backbone.js\n       </strong>\n      </a>\n      to new heights.\n      <strong>\n       Giraffe\n      </strong>\n      \'s goal is to follow the\n      <strong>\n       Backbone\n      </strong>\n      philosophy of unopinionated\nsimplicity to provide commonly needed features with few assumptions. It differs\nfrom other\n      <strong>\n       Backbone\n      </strong>\n      libraries like\n      <strong>\n       Marionette\n      </strong>\n      and\n      <strong>\n       Chaplin\n      </strong>\n      in its\nreduced scope and size, and it takes a different approach to the problems of\nroute handling, object lifecycles, event aggregation, and view management.\n     </p>\n     <h2 id="h-overview">\n      Overview\n     </h2>\n     <ul>\n      <li>\n       <p>\n        <strong>\n         Giraffe.View\n        </strong>\n        is a nestable and flexible class that provides lifecycle\nmanagement and many other useful features. It defaults to\n        <strong>\n         Underscore\n        </strong>\n        templates and easily supports any form of templating.\n       </p>\n      </li>\n      <li>\n       <p>\n        <strong>\n         Giraffe.App\n        </strong>\n        is a special view that helps your views, models, collections,\nand routers communicate by acting as an event hub. Multiple apps can coexist and\nteardown is as simple as it gets.\n       </p>\n      </li>\n      <li>\n       <p>\n        <strong>\n         Giraffe.Router\n        </strong>\n        leverages an app\'s events to trigger routing events that\nany object can listen for. It also has reverse routes to allow the construction\nof URLs using app events and arguments.\n       </p>\n      </li>\n      <li>\n       <p>\n        <strong>\n         Giraffe.Model\n        </strong>\n        and\n        <strong>\n         Giraffe.Collection\n        </strong>\n        are thin wrappers that add\n        <strong>\n         Giraffe\n        </strong>\n        \'s lifecycle management and app events. Any object can mix\nin this functionality via\n        <code>\n         Giraffe.configure\n        </code>\n        .\n       </p>\n      </li>\n     </ul>\n     <h2 id="h-documentation">\n      Documentation\n     </h2>\n     <p>\n      Read the\n      <a href="http://barc.github.io/backbone.giraffe/backbone.giraffe.html">\n       <strong>\n        API docs\n       </strong>\n      </a>\n      and\ncheck out our\n      <a href="http://barc.github.io/backbone.giraffe/viewBasics.html">\n       <strong>\n        live examples\n       </strong>\n      </a>\n      .\n     </p>\n     <h2 id="h-how-giraffe-is-different">\n      How Giraffe is Different\n     </h2>\n     <p>\n      <strong>\n       Giraffe\n      </strong>\n      was created by the needs of our team as we built\n      <a href="http://barc.com">\n       <strong>\n        Barc\n       </strong>\n      </a>\n      . We tried many existing libraries but some did way too\nmuch, others added too many layers, and others performed poorly.\n     </p>\n     <p>\n      <strong>\n       Giraffe\n      </strong>\n      does not have all the bells and whistles of the larger frameworks.\nWe found the effort to customize them for our needs was more effort than simply\nbuilding upon\n      <strong>\n       Backbone\n      </strong>\n      with a minimalist approach. For example, there is no\nconcept of specialized containers like regions or layouts, as any view in\n      <strong>\n       Giraffe\n      </strong>\n      can act as a parent of one or more child views.\n      <strong>\n       Giraffe\n      </strong>\n      also\nhas no CollectionView or ItemView\n(see\n      <a href="http://barc.github.io/backbone.giraffe/collectionView.html">\n       <strong>\n        Giraffe.Contrib.CollectionView\n       </strong>\n      </a>\n      and\n      <a href="http://barc.github.io/backbone.giraffe/fastCollectionView.html">\n       <strong>\n        Giraffe.Contrib.FastCollectionView\n       </strong>\n      </a>\n      ),\nbut we are open to suggestions to make\n      <strong>\n       Giraffe\n      </strong>\n      as useful as possible to\n      <strong>\n       Backbone\n      </strong>\n      developers who want an extension library with few opinions.\n     </p>\n     <p>\n      Is this framework for you? It depends. We feel\n      <strong>\n       Giraffe\n      </strong>\n      adds essential\nfeatures to make you more productive with\n      <strong>\n       Backbone\n      </strong>\n      .\n     </p>\n     <h3 id="h-highlights">\n      Highlights\n     </h3>\n     <ul>\n      <li>\n       <p>\n        <a href="http://barc.github.io/backbone.giraffe/routersAndAppEvents.html">\n         <strong>\n          Routes emit events\n         </strong>\n        </a>\n        instead of being tied to functions. This makes it extremely simple for a deeply\nnested view to act on a route.\n       </p>\n      </li>\n      <li>\n       <p>\n        <a href="http://barc.github.io/backbone.giraffe/backbone.giraffe.html#Router">\n         <strong>\n          Reverse routes with arguments\n         </strong>\n        </a>\n        provide a way to trigger routes in the application using app events without\nhaving to know a URL path.\n       </p>\n      </li>\n      <li>\n       <p>\n        <a href="http://barc.github.io/backbone.giraffe/appEvents.html">\n         <strong>\n          Giraffe.App\n         </strong>\n        </a>\n        is a\nspecial view that acts as an event hub to help your app communicate and respond\nto routes, and all\n        <strong>\n         Giraffe\n        </strong>\n        objects have convenient\n        <code>\n         appEvents\n        </code>\n        bindings\ninspired by\n        <code>\n         Backbone.View#events\n        </code>\n        .\n       </p>\n      </li>\n      <li>\n       <p>\n        <a href="http://barc.github.io/backbone.giraffe/backbone.giraffe.html#View-attachTo">\n         <strong>\n          <code>\n           Giraffe.View#attachTo(someElement)\n          </code>\n         </strong>\n        </a>\n        allows views to move anywhere on the DOM without clobbering each other\'s events,\nand it automatically sets up parent-child relationships for memory management.\n       </p>\n      </li>\n      <li>\n       <p>\n        <a href="http://barc.github.io/backbone.giraffe/lifecycleManagement.html">\n         <strong>\n          Lifecycle management\n         </strong>\n        </a>\n        mitigates memory leaks. It\'s automatic for nested views and can be used for any\nobject with a\n        <code>\n         dispose\n        </code>\n        method via\n        <code>\n         Giraffe.View#addChild\n        </code>\n        .\n       </p>\n      </li>\n      <li>\n       <p>\n        <a href="http://barc.github.io/backbone.giraffe/appInitialization.html">\n         <strong>\n          (A)sync app initialization\n         </strong>\n        </a>\n        helps an app reach its ready state. For example, an app may need to wait for\nasynchronous bootstrap data or a websocket connection before starting.\n       </p>\n      </li>\n      <li>\n       <p>\n        <a href="http://barc.github.io/backbone.giraffe/documentEvents.html">\n         <strong>\n          Declarative event handling\n         </strong>\n        </a>\n        in markup provides simple one-way binding. (does not try to be\n        <strong>\n         Knockout\n        </strong>\n        or\n        <strong>\n         AngularJS\n        </strong>\n        )\n       </p>\n      </li>\n      <li>\n       <p>\n        <a href="https://github.com/barc/backbone.giraffe/blob/master/src/backbone.giraffe.contrib.coffee">\n         <strong>\n          Optional Giraffe.Contrib\n         </strong>\n        </a>\n        extensions include a simple Controller class with\n        <strong>\n         Giraffe\n        </strong>\n        \'s lifecycle and\n        <code>\n         appEvents\n        </code>\n        features,\na\n        <a href="http://barc.github.io/backbone.giraffe/collectionView.html">\n         CollectionView\n        </a>\n        that renders a collection with a view per model,\nand
