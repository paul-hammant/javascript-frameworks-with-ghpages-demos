id="functional-and-reactive">\n      Functional and Reactive\n     </h2>\n     <p>\n      Functional enables \xe2\x80\x9cpredictable\xe2\x80\x9d code, and Reactive enables \xe2\x80\x9cseparated\xe2\x80\x9d code. Cycle.js apps are made of pure functions, which means you know they only take inputs and generate predictable outputs, without performing any I/O effects. The building blocks are reactive streams from libraries like\n      <a href="http://staltz.com/xstream">\n       xstream\n      </a>\n      ,\n      <a href="http://reactivex.io/rxjs">\n       RxJS\n      </a>\n      or\n      <a href="https://github.com/cujojs/most/">\n       Most.js\n      </a>\n      , which greatly simplify code related to events, asynchrony, and errors. Structuring the application with streams also separates concerns, because all\n      <a href="/observables.html#reactive-programming">\n       dynamic updates to a piece of data are co-located\n      </a>\n      and impossible to change from outside. As a result, apps in Cycle are entirely\n      <code class="highlighter-rouge">\n       this\n      </code>\n      -less and have nothing comparable to imperative calls such as\n      <code class="highlighter-rouge">\n       setState()\n      </code>\n      or\n      <code class="highlighter-rouge">\n       update()\n      </code>\n      .\n     </p>\n    </div>\n    <div class="homepage-features">\n     <h2 id="simple-and-concise">\n      Simple and Concise\n     </h2>\n     <p>\n      Cycle.js is a framework with very few concepts to learn. The core API has just one function:\n      <code class="highlighter-rouge">\n       run(app, drivers)\n      </code>\n      . Besides that, there are\n      <strong>\n       streams\n      </strong>\n      ,\n      <strong>\n       functions\n      </strong>\n      ,\n      <strong>\n       drivers\n      </strong>\n      (plugins for different types of I/O effects), and a helper function to isolate scoped components. This is a framework with very little \xe2\x80\x9cmagic\xe2\x80\x9d. Most of the building blocks are just JavaScript functions. Usually the lack of \xe2\x80\x9cmagic\xe2\x80\x9d leads to very verbose code, but since functional reactive streams are able to build complex dataflows with a few operations, you will come to see how apps in Cycle.js are small and readable.\n     </p>\n    </div>\n    <div class="homepage-features">\n     <h2 id="extensible-and-testable">\n      Extensible and Testable\n     </h2>\n     <p>\n      Drivers are plugin-like simple functions that take messages from sinks and call imperative functions. id="functional-and-reactive">\n      Functional and Reactive\n     </h2>\n     <p>\n      Functional enables \xe2\x80\x9cpredictable\xe2\x80\x9d code, and Reactive enables \xe2\x80\x9cseparated\xe2\x80\x9d code. Cycle.js apps are made of pure functions, which means you know they only take inputs and generate predictable outputs, without performing any I/O effects. The building blocks are reactive streams from libraries like\n      <a href="http://staltz.com/xstream">\n       xstream\n      </a>\n      ,\n      <a href="http://reactivex.io/rxjs">\n       RxJS\n      </a>\n      or\n      <a href="https://github.com/cujojs/most/">\n       Most.js\n      </a>\n      , which greatly simplify code related to events, asynchrony, and errors. Structuring the application with streams also separates concerns, because all\n      <a href="/observables.html#reactive-programming">\n       dynamic updates to a piece of data are co-located\n      </a>\n      and impossible to change from outside. As a result, apps in Cycle are entirely\n      <code class="highlighter-rouge">\n       this\n      </code>\n      -less and have nothing comparable to imperative calls such as\n      <code class="highlighter-rouge">\n       setState()\n      </code>\n      or\n      <code class="highlighter-rouge">\n       update()\n      </code>\n      .\n     </p>\n    </div>\n    <div class="homepage-features">\n     <h2 id="simple-and-concise">\n      Simple and Concise\n     </h2>\n     <p>\n      Cycle.js is a framework with very few concepts to learn. The core API has just one function:\n      <code class="highlighter-rouge">\n       run(app, drivers)\n      </code>\n      . Besides that, there are\n      <strong>\n       streams\n      </strong>\n      ,\n      <strong>\n       functions\n      </strong>\n      ,\n      <strong>\n       drivers\n      </strong>\n      (plugins for different types of I/O effects), and a helper function to isolate scoped components. This is a framework with very little \xe2\x80\x9cmagic\xe2\x80\x9d. Most of the building blocks are just JavaScript functions. Usually the lack of \xe2\x80\x9cmagic\xe2\x80\x9d leads to very verbose code, but since functional reactive streams are able to build complex dataflows with a few operations, you will come to see how apps in Cycle.js are small and readable.\n     </p>\n    </div>\n    <div class="homepage-features">\n     <h2 id="extensible-and-testable">\n      Extensible and Testable\n     </h2>\n     <p>\n      Drivers are plugin-like simple functions that take messages from sinks and call imperative functions. All I/O effects are contained in drivers. This means your application is just a pure function, and it becomes easy to swap drivers around. The community has built drivers for\n      <a href="https://github.com/cyclejs/cycle-react-native">\n       React Native\n      </a>\n      ,\n      <a href="https://github.com/cyclejs/cycle-notification-driver">\n       HTML5 Notification\n      </a>\n      ,\n      <a href="https://github.com/cgeorg/cycle-socket.io">\n       Socket.io\n      </a>\n      , etc. Sources and sinks can be easily used as\n      <a href="https://iancooper.github.io/Paramore/ControlBus.html">\n       Adapters and Ports\n      </a>\n      . This also means testing is mostly a matter of feeding inputs and inspecting the output. No deep mocking needed. Your application is just a pure transformation of data.\n     </p>\n    </div>\n    <h2 id="explicit-dataflow">\n     Explicit dataflow\n    </h2>\n    <p>\n     In every Cycle.js app, each of the stream declarations is a node in a dataflow graph, and the dependencies between declarations are arrows.
