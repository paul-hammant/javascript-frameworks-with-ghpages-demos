<section>\n     <h4>\n      Installation\n     </h4>\n     <p>\n      Download Angulartics and the Angulartics plugin for your vendor. Note: All plugins list Angulartics as a dependency, so they will be downloaded together.\n     </p>\n     <pre><code>$ bower install <strong class="highlighted">angulartics-google-analytics</strong> --save</code></pre>\n     <p>\n      Load the relevant files in your applications html (or include them in your build script).\n     </p>\n     <pre><code>&lt;script src="<strong class="highlighted">/bower_components/angulartics/dist/angulartics.min.js</strong>"&gt;&lt;/script&gt;\n&lt;script src="<strong class="highlighted">/bower_components/angulartics-google-analytics/dist/angulartics-ga.min.js</strong>"&gt;&lt;/script&gt;\n</code></pre>\n     <p>\n      In your application dependencies, inject Angulartics and the Angulartics plugin for your vendor.\n     </p>\n     <pre><code>angular.module(\'myApp\', [\'<strong class="highlighted">angulartics</strong>\', \'angulartics.<strong class="highlighted">google.analytics</strong>\'])</code></pre>\n     <div class="alert alert-info">\n      <p>\n       Make sure you\n       <strong>\n        delete\n       </strong>\n       any automatic tracking line from your vendor snippet code!\n      </p>\n      <br/>\n      <pre><code>// Google Analytics example\n<strong><del>ga(\'send\', \'pageview\');</del> // &lt;---- delete this line!</strong></code></pre>\n     </div>\n    </section>\n    <br/>\n    <section id="page-tracking">\n     <h4>\n      Virtual pageviews\n     </h4>\n     <p>\n      Angulartics does automatic virtual pageview tracking by default, meaning the entire\n        user navigation across the different routes of your application is tracked. You can turn it\n        off with\n      <code>\n       virtualPageviews(false)\n      </code>\n      .\n     </p>\n     <pre><code>module.config(function (<strong class="highlighted">$analyticsProvider</strong>) {\n  // turn off automatic tracking\n  $analyticsProvider.<strong class="highlighted">virtualPageviews</strong>(<strong>false</strong>);</code></pre>\n    </section>\n    <br/>\n    <section id="event-tracking">\n     <h4>\n      Declarative event tracking\n     </h4>\n     <p>\n      The simplest way to do event tracking is by adding the attributes\n      <code>\n       analytics-on\n      </code>\n      and\n      <code>\n       analytics-event\n      </code>\n      to the element. Think of it as:\n      <strong>\n       <i>\n        on\n       </i>\n      </strong>\n      \'click\',\n      track\n      <strong>\n       <i>\n        event\n       </i>\n      </strong>\n      \'name\'.\n     </p>\n     <pre><code>&lt;a href="file.pdf" <strong class="highlighted">analytics-on</strong>=<strong>"click"</strong> <strong class="highlighted">analytics-event</strong>=<strong>"Download"</strong>&gt;Download&lt;/a&gt;</code></pre>\n     <div class="alert alert-info">\n      Depending on your analytics provider, the\n      <code>\n       analytics-event\n      </code>\n      attribute is automatically\n      mapped to its respective vendor field. For example, with\n      <strong>\n       Google Analytics\n      </strong>\n      , the event is used for\n      the \'action\' parameter.\n     </div>\n    </section>\n    <br/>\n    <section>\n     <h4>\n      Inferred values\n     </h4>\n     <p>\n      <code>\n       analytics-on\n      </code>\n      always needs to be present, because it is the attribute that enables\n      event tracking for the element. All other attributes will be inferred if omitted.\n     </p>\n     <pre><code>&lt;!-- infers <strong>analytics-event</strong> from the element inner text --&gt;\n&lt;a href="file.pdf" analytics-on="click"&gt;<strong class="highlighted">Download</strong>&lt;/a&gt;\n\n&lt;!-- infers the <strong>analytics-on</strong> value from the element type; for anchor is \'click\' --&gt;\n&lt;<strong class="highlighted">a</strong> href="file.pdf" analytics-on&gt;Download&lt;/a&gt;\n\n&lt;!-- same as adding <strong>analytics-event</strong>="Step 3" --&gt;\n&lt;input type="submit" value="<strong class="highlighted">Step 3</strong>" analytics-on&gt;</code></pre>\n    </section>\n    <br/>\n    <section>\n     <h4>\n      Additional parameters\n     </h4>\n     <p>\n      Each vendor has its own set of tracking properties. For example,\n      <strong>\n       Google Analytics\n      </strong>\n      have\n      <i>\n       category\n      </i>\n      and\n      <i>\n       label\n      </i>\n      .\n      You can set these properties with\n      <code>\n       analytics-\n      </code>\n      attributes.\n     </p>\n     <pre><code>&lt;button analytics-on analytics-event="Play" <strong class="highlighted">analytics-category</strong>="<strong>Videos</strong>" <strong class="highlighted">analytics-label</strong>="<strong>Gone with the Wind</strong>"&gt;Play&lt;/button&gt;</code></pre>\n    </section>\n    <br/>\n    <section id="scroll-tracking">\n     <h4>\n      Scroll tracking\n     </h4>\n     <p>\n      In the same way you can track an action by catching any valid DOM event (\n      <em>\n       click\n      </em>\n      ,\n      <em>\n       hover\n      </em>\n      , etc), you can also\n      track any element that appears into the viewport while scrolling the page. To achieve this, Angulartics uses\n      <strong>\n       <a href="http://imakewebthings.com/jquery-waypoints/">\n        jQuery Waypoints\n       </a>\n      </strong>\n      and provides you with a\n      <strong>\n       <em>\n        scrollby\n       </em>\n      </strong>\n      event that you can use for\n      <code>\n       analytics-on\n      </code>\n      .\n     </p>\n     <p>\n      First, you need to include the plugin script named\n      <code>\n       angulartics-scroll.js\n      </code>\n      , and inject\n      <code>\n       angulartics.scroll\n      </code>\n      into your module.\n     </p>\n     <pre><code>angular.module(\'myApp\', [..., \'angulartics.<strong class="highlighted">scroll</strong>\'])</code></pre>\n     <pre><code>&lt;section id="features" analytics-on="<strong class="highlighted">scrollby</strong>"&gt;</code></pre>\n     <p>\n      Remember what we learned about inferred values: in the example above we didn\'t specify an event name so in this case\n      the\n      <code>\n       id\n      </code>\n      attribute ("features") will be used.\n     </p>\n     <div class="alert alert-info">\n      <p>\n       <strong>\n        jQuery Waypoints\n       </strong>\n       have\n       <a href="http://imakewebthings.com/jquery-waypoints/#doc-options">\n        a set of options\n       </a>\n       to customize its behavior; you can specify any of them by adding\n       <code>\n        scrollby-*\n       </code>\n       attributes. For more information you can take a look at\n       <a href="http://github.com/angulartics/angulartics/blob/master/samples/scroll.html">\n        this sample\n       </a>\n       .\n      </p>\n     </div>\n    </section>\n    <br/>\n    <section>\n     <h4>\n      Using the API\n     </h4>\n     <p>\n      In order to track pageviews and events from within your application logic,\n      inject\n
